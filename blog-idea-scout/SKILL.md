週1回、daily-news が蓄積したニュースと Obsidian Vault（外部脳）を突き合わせ、**nobu666.com に書く価値のあるブログ記事ネタを提案**する。提案は別ファイルに追記し、本人が手動で本採用する。

## 大前提：質を量より優先する（最重要）

ニュース→記事ネタは大半がノイズ。書く価値があるのは稀。**むりやり数をひねり出さない**。

- 提案は **0〜3件**。良いものが無ければ **「今週は該当なし」とだけ書いて終了**してよい（むしろそれが普通）
- 弱い「トレンドだから書け」は出さない。**本人の既存メモ・過去記事・関心に繋がるニュースだけ**拾う
- nobu666.com は考察・体験ベースの個人ブログ。ニュースの再要約は記事にならない。「本人の蓄積（外部脳）と新ニュースが結びついて初めて記事になる」を判断基準にする

## 手順

### 1. 直近1週間のニュースを読む

`date` で過去7日分のファイル名を割り出し、`~/Documents/Obsidian/Vault/News/` の `YYYY-MM-DD-morning.md` / `-evening.md` を Read で開く（存在しないものはスキップ）。

- **反復＝シグナル**：複数日にわたって繰り返し出てくるトピックを重視する（一過性の単発より、継続して話題になっているテーマの方が記事の芯になりやすい）

### 2. 外部脳と既存ネタを読む

- `~/Documents/Obsidian/Vault/ブログ記事ネタ帳.md` を読み、**既存ネタ（ネタ1〜）と重複するものは提案しない**
- `~/repos/obsidian-vault-search/vault-search index >/dev/null 2>&1` で索引更新後、`~/repos/obsidian-vault-search/vault-search "<ニュースに関連する自然文クエリ>"` で、今週のニュースに関連する**過去メモ・過去記事**を掘る（Ollama 未起動ならスキップ）
- 公開済み記事（`~/Documents/Obsidian/Vault/ブログ記事ネタ帳.md` の公開済みエントリや `content/` の既存記事）と内容がかぶるネタは出さない

### 3. ネタ候補を組み立てる

各候補は「**新しいニュース × 本人の既存蓄積**」の掛け算で作る。良い候補の条件:

- 今週のニュースが、本人の過去メモ／過去記事／繰り返しの関心に**結びついている**
- 「へぇ」で終わらず、読者の手元が変わる・考えが進む角度がある
- 既存ネタ帳の在庫と重複していない

弱い候補（単なるニュース要約、本人の蓄積に繋がらないトレンド紹介）は**捨てる**。

### 4. 出力（別ファイルに追記）

保存は必ず **Write ツール**で行う（親ディレクトリは自動作成。無人実行では Bash でのファイル作成は権限ダイアログで止まるため使わない）。

出力先: `~/Documents/Obsidian/Vault/ブログ記事ネタ_AI提案.md`

**既存内容があれば末尾に追記**（既存を消さない。Write前に必ずRead）。先頭に無ければ見出し `# ブログ記事ネタ AI提案（blog-idea-scout）` を付ける。

今週分のセクションを以下の形式で足す:

```
## YYYY-MM-DD 週次提案

（候補がある場合、1〜3件）

### 候補: <記事タイトル案>
- **掴み**: 読者を引き込む切り口を1〜2行
- **芯**: 何を論じる記事か。なぜ本人が書く価値があるか（既存メモ/過去記事との繋がりを明示。`[[ノート名]]` でリンク）
- **発端ニュース**: 該当ニュースの個別記事URL（複数可）
- **刺さる層**: 誰に届くか一言

（候補が無い場合）
今週は該当なし。（理由を一言: 例「目ぼしいニュースは出たが本人の蓄積に結びつく角度が無かった」）
```

## 注意事項

- これはあくまで**提案の受信箱**。手で育てる `ブログ記事ネタ帳.md` には書き込まない（本人が良いものだけ手動で昇格する）
- ニュースのURLは実在する**個別記事**であること（一覧/インデックスページを貼らない）
- 提案ゼロを失敗と捉えない。質の低い在庫を増やす方が有害
